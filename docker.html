数据挂载

docker提供两种将数据从宿主机挂载到容器中：

  • volumes:将docker的文件系统挂载到/var/lib/docker/volumes
  • bind mounts:将宿主机上的任意位置的文件或目录挂载到容器中

volumes示例：

  1. 创建数据卷
    docker volume create nginx-vol
docker volume ls
docker volume inspect nginx-vol
  2. 使用数据卷
    docker run -d --name=nginx-test --mount src=nginx-vol,dst=/usr/share/nginx/html nginx
docekr run -d --name=nginx-test -v nginx-vol:/usr/share/nginx/html nginx

bind mount示例：

  1. 挂载宿主机目录到容器
    docker run -d --name=nginx-test --mount type=bind,src
docker volume ls
docker volume inspect nginx-vol
  2. 使用数据卷
    docker run -d --name=nginx-test --mount src=nginx-vol,dst=/usr/share/nginx/html nginx
docekr run -d --name=nginx-test -v nginx-vol:/usr/share/nginx/html nginx

Docker Compose使用

Docker Compose使用YAML文件来定义多服务的应用。YAML是JSON的一个子集。

Docker Compose默认使用文件名docker-compose.yml，用户也可以使用-f参数指定具体文件。

Compose文件示例：

# 使用Compose文件部署一个小型Flask应用
# 文件的基本结构包含四个一级Key: version、services、networks、volumes
version: '3.5' # 必须指定，定义了Compose文件格式(主要是API)，并非是docker等软件版本，也不是镜像版本
services: # 定义不同的应用服务，并使用对应key去创建容器
  web-fe: # web前端服务
    build: . # 指定Docker基于当前目录(.)下的Dockerfile中的定义指令来构建一个新镜像，这个镜像会被用于启动该服务的容器
    command: python app.py # 指定Docker容器中名为app.py的Python脚本作为主程序吗，当然，此镜像中必须包括app.py文件以及Python(Dockerfile) 可以覆盖Dockerfile中的CMD命令
    ports: # 定义端口
      - target: 5000 # 容器内端口
        published: 5000 # 发布端口=宿主机端口
    networks: # 将web连接到指定的网络上
      - counter-net
    volumes: # 指定数据卷 
      - type: volume # 数据卷类型
        source: counter-volume # 数据卷(宿主机)
        target: /code # 容器内路径
  redis: # redis服务
    image: "redis:alpine" # 指定镜像，该镜像会从Docker Hub中拉取下来
    networks: # 将redis连接到指定网络上
      counter-net:

networks: # 用于指引Docker创建新的网络
  counter-net:

volumes: # 用于指引Docker创建新的数据卷
  counter-vol:

构建springboot镜像的Dockerfile示例：

FROM openjdk:8
MAINTAINER intbjw
ADD ./target/dockerdemo-0.0.1-SNAPSHOT.jar app.jar
VOLUME /code
EXPOSE 8080
RUN bash -c 'touch /app.jar'
# 用于设定容器启动时第一个运行的命令及器参数
ENTRYPOINT ["java","-Djava.security.egd=file:/dev/./urandom","-jar","/app.jar"]

How to create a DB for MongoDB container on start up?

The docker hub image will run any scripts in /docker-entrypoint-initdb.d/ when there is nothing populated in the /data/db directory. The mongo container image provides the /docker-entrypoint-initdb.d/ path to deploy custom .js or .sh setup scripts that will be run once on database initialisation.

https://stackoverflow.com/questions/42912755/how-to-create-a-db-for-mongodb-container-on-start-up

docker-compose.yml

version: '3.8'
services:
  mongodb:
    image: mongo
    hostname: mongodb
    command: [--auth]
    environment:
      MONGO_INITDB_ROOT_USERNAME: root
      MONGO_INITDB_ROOT_PASSWORD: password
      MONGO_INITDB_DATABASE: database
      MONGO_USERNAME: username
      MONGO_PASSWORD: password
    volumes:
      - ./.docker/mongo_data:/data/db
      - ./mongo-init.sh:/docker-entrypoint-initdb.d/mongo-init.sh

mongo-init.sh

mongo -- "$MONGO_INITDB_DATABASE" <<EOF
db = db.getSiblingDB('admin')
db.auth('$MONGO_INITDB_ROOT_USERNAME', '$MONGO_INITDB_ROOT_PASSWORD')
db = db.getSiblingDB('$MONGO_INITDB_DATABASE')
db.createUser({
  user: "$MONGO_USERNAME",
  pwd: "$MONGO_PASSWORD",
  roles: [
  { role: 'readWrite', db: '$MONGO_INITDB_DATABASE' }
  ]
})
EOF

使用Docker部署Tomcat

dockerfile

FROM tomcat:8.5

#将webapp下的全部删除
RUN rm -rf /usr/local/tomcat/webapps/* \
&& useradd -ms /bin/bash tomcat \
&& chown -R tomcat:tomcat /usr/local/tomcat/
RUN chmod 777 /usr/local/tomcat/bin/catalina.sh 
#将target下的xx.war拷贝到/usr/local/tomcat/webapps/下
ADD --chown=tomcat ROOT.war /usr/local/tomcat/webapps/
# 设置启动用户
USER tomcat
# 设置工作目录
WORKDIR /usr/local/tomcat
#端口
EXPOSE 8080
#设置启动命令
ENTRYPOINT ["/usr/local/tomcat/bin/catalina.sh","run"]
